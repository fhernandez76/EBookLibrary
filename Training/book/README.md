# EBook Library — Book Edition Build System

This folder produces the **Book Edition** of the EBook Library training guide:
a professional, book-ready volume aimed at junior-to-lead full-stack engineers.

The original `Training/EBookLibrary-Training-Guide.docx` is **not modified**;
all artifacts here are independent and live under `dist/`.

---

## Folder layout

```
book/
├── build-book.js             # main builder (DOCX + EPUB + optional PDF)
├── package.json              # adm-zip + mermaid-filter
├── README.md                 # this file
│
├── src/                      # manuscript (Markdown), authored for book voice
│   ├── frontmatter/          # cover, copyright, preface, ...
│   ├── part-1-foundations/   # ch 1-3
│   ├── part-2-implementation/# ch 4-11
│   ├── part-3-architecture-scale/  # ch 12-15
│   └── backmatter/           # appendices, glossary, bibliography, index
│
├── assets/
│   ├── figures/              # JPGs (copied from docs/architecture/images + screenshots)
│   └── styles/
│       ├── reference.docx    # generated by create-reference.js
│       ├── create-reference.js
│       ├── callouts.lua      # Pandoc Lua filter (sidebar callouts)
│       └── epub.css
│
├── tmp/                      # combined Markdown (regenerated each build)
└── dist/                     # output artifacts
```

## Build

```powershell
# from book/
node build-book.js
```

Outputs (depending on installed engines):

| File                                  | Always |
|---------------------------------------|--------|
| `dist/EBookLibrary-Training-Guide-v2.docx` | ✓     |
| `dist/EBookLibrary-Training-Guide-v2.epub` | ✓     |
| `dist/EBookLibrary-Training-Guide-v2.pdf`  | only if a PDF engine is found |

### Flags

```powershell
node build-book.js --docx-only     # DOCX only
node build-book.js --no-pdf        # skip the PDF attempt
```

## Toolchain

| Tool          | Required | Purpose                             | Verified version |
|---------------|----------|-------------------------------------|------------------|
| pandoc        | yes      | Markdown → DOCX / EPUB / PDF        | 3.9.0.2          |
| node          | yes      | Build script + reference.docx patch | v22.20.0         |
| adm-zip       | yes      | DOCX is a ZIP — used to patch styles | npm              |
| mermaid-filter| optional | Render leftover Mermaid blocks      | npm (installed)  |
| xelatex *or* weasyprint | optional | PDF generation              | not installed    |

### PDF on Windows — three options

1. **Easiest.** Open the generated DOCX in Microsoft Word and choose
   *File ▸ Save As ▸ PDF*. Highest fidelity, no extra tooling.
2. Install **MiKTeX** (https://miktex.org) which provides `xelatex`.
   Then re-run `node build-book.js`.
3. Install **WeasyPrint + GTK runtime**
   (https://doc.courtbouillon.org/weasyprint/stable/first_steps.html)
   and re-run.

## Editorial conventions used in this book

The Lua filter `assets/styles/callouts.lua` recognizes four sidebar styles
unique to the Book Edition. Author them in Markdown like this:

```markdown
> **Foundations:** Extra background for readers new to the topic.

> **Architect's Note:** Trade-offs, alternatives, and "when not to use".

> **Pitfall:** A common mistake and how to avoid it.

> **In Practice:** How real engineering teams actually apply this.
```

Plus the legacy callouts (`Tip`, `Note`, `Warning`, `Important`,
`Checkpoint ✅` headings, and `🤖` AI-assisted headings) are still honored
for back-compat with the chapter sources copied from `Training/`.

### Numbered figures, listings, and tables

Captions are authored manually in the manuscript (no `pandoc-crossref`
dependency on Windows). Conventions:

- **Figure** caption: place an image alone in a paragraph; the alt text
  becomes the figure caption (e.g. `![Figure 2.1 — Clean Architecture layers](figures/05-clean-architecture-layers.jpg)`).
- **Listing** caption: put a one-line bold paragraph immediately above
  the code block (e.g. `**Listing 4.2 — Author entity.**`).
- **Table** caption: put a one-line bold paragraph immediately above the
  pipe table (e.g. `**Table 7.1 — Standard response envelope fields.**`).
