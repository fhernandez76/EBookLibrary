# Acknowledgments

A book that describes a working software project is the work of more than
its named author.

Thanks to the engineers who reviewed early drafts of the training guide
that became this book, asked the questions that surfaced gaps, and pushed
back on the parts that read like cleverness instead of clarity.

Thanks to the open-source maintainers behind .NET, ASP.NET Core, Entity
Framework Core, MediatR, FluentValidation, AutoMapper, React, Vite,
Tailwind CSS, Zustand, React Query, Blazor, xUnit, Moq, FluentAssertions,
Playwright, Pandoc, and the countless smaller libraries that this project
quietly depends on. Each of those projects represents thousands of hours of
unpaid labor by people who will never read this acknowledgment.

Thanks to the AI coding assistants that served simultaneously as the
*subject* of Chapter 14 and a working *tool* in the writing of every other
chapter. The conversation about how to use them well is one this book
hopes to contribute to.

Finally, thanks to the readers. A book about software has the unusual
property that the reader's hands on a keyboard finish the work the author
started. The architecture only matters if it is used, the patterns only
matter if they are applied, and the decisions only matter if they are
revisited. That part of the book is yours to write.

— *Francisco Hernandez, 2026*
